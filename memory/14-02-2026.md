### âœ… Fixed Message Duplication Bug
**Problem:** Messages were appearing twice in FLOW dashboard chat  
**Root cause:** SSE handler was directly appending messages to DOM (`chatMessages.appendChild()`), while polling also fetched and re-rendered all messages via `renderMessages()`. When both fired, duplicates appeared.

**Fix:** Changed SSE `onmessage` handler to call `loadChat()` instead of direct DOM manipulation. This ensures single source of truth (server) and prevents race condition duplicates.

**File:** `app/dashboard.html` (line ~3763)

**Before:**
```javascript
eventSource.onmessage = (event) => {
    const message = JSON.parse(event.data);
    // Skip connection message
    if (message.type === 'connected') return;
    
    // Add message to chat immediately - BUG: duplicates with polling
    const messageDiv = document.createElement('div');
    // ... build message HTML ...
    chatMessages.appendChild(messageDiv);
}
```

**After:**
```javascript
eventSource.onmessage = (event) => {
    const message = JSON.parse(event.data);
    if (message.type === 'connected') return;
    
    // Refresh from server - polling keeps it synced, no duplicates
    loadChat();
}
```
